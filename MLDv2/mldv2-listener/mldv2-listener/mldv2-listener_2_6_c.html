Test Label

MLD.2.6: MLDv2 Query Message Verification - Payload Length
Part B: Payload Length is too short

Purpose

To verify that MLDv2 Listener properly processes Payload Length in MLDv2 Query message.

References

RFC3810 5.1.12. Additional Data
RFC3810 8.1. Query Version Distinctions

Discussion

None

Test Requirement

None

  • Common Test Setup
    1. Configure HUT to have:
      • Robustness Variable of 2
      • Query Interval of 125
      • Query Response Interval of 10 seconds
      • Unsolicited Report Interval of 1 second

  • Common Test Cleanup
    1. Return all timers and variables to their defaults values.
    2. Stop any listening. HUT does not listen to any multicast other than a solicited-node multicast address of an address assigned to the interface.

Procedure

Step 11. Enable MLDv2 on the HUT.
Step 12. Call ListenerAPI (S1, I1, M1, INCLUDE, {S1}) on the HUT.
Step 13. Observe the packets on all networks.
Step 14. TR1 transmits MLDv2 Query message C.
    -----
    IPv6 Header
        Payload Length                  = 52 (unusual)
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TR1 link-local address
        IPv6 Destination Address        = All Nodes Address
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLD Query)
        Maximum Response Code           = 10000
        Multicast Address               = ::
        S Flag                          = 0
        QRV                             = 2
        QQIC                            = 125
        Number of Sources               = 0
    -----
Step 15. Observe the packets on all networks.

Observable Results

Step 13: The HUT must transmit 2 MLDv2 Report messages with Multicast Address Record ALLOW (S1) for M1.
Step 15: The HUT must not transmit MLDv2 Report message with Multicast Address Record for M1.
