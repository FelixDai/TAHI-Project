Test Label

MLD.4.4: MLDv2 Multicast Address Specific Query in Multiple Listening State
Part C: EXCLUDE (A) for M1 and EXCLUDE (NULL) for M2

Purpose

To verify that the MLDv2 Listener replies multiple listening states when the MLDv2 Multicast Address Specific Query message is received.

References

RFC3810 6.2. Action on Reception of a Query
RFC3810 6.3. Action on Timer Expiration

Discussion

None

Test Requirement

None

  • Common Test Setup
    1. Configure HUT to have:
      • Robustness Variable of 2
      • Query Interval of 125
      • Query Response Interval of 10 seconds
      • Unsolicited Report Interval of 1 second

  • Common Test Cleanup
    1. Return all timers and variables to their defaults values.
    2. Stop any listening. HUT does not listen to any multicast other than a solicited-node multicast address of an address assigned to the interface.

Procedure

Step 19. Enable MLDv2 on the HUT.
Step 20. Call ListenerAPI (S1, I1, M1, EXCLUDE, {S1}) on the HUT.
Step 21. Observe the packets on all networks.
Step 22. Call ListenerAPI (S2, I1, M2, EXCLUDE, {NULL}) on the HUT.
Step 23. Observe the packets on all networks.
Step 24. TR1 transmits MLDv2 Multicast Address Specific Query message A for M1.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TR1 Link-local Address
        IPv6 Destination Address        = M1
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLD Query)
        Maximum Response Code           = 1000
        Multicast Address               = M1
        S Flag                          = 0
        QRV                             = 2
        QQIC                            = 125
        Number of Sources               = 0
    -----
Step 25. Observe the packets on all networks.
Step 26. TR1 transmits MLDv2 Multicast Address Specific Query message B for M2.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TR1 Link-local Address
        IPv6 Destination Address        = M2
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLD Query)
        Maximum Response Code           = 1000
        Multicast Address               = M2
        S Flag                          = 0
        QRV                             = 2
        QQIC                            = 125
        Number of Sources               = 0
    -----
Step 27. Observe the packets on all networks.

Observable Results

Step 21: The HUT must transmit 2 MLDv2 Report messages with Multicast Address Record TO_EX (S1) for M1.
Step 23: The HUT must transmit 2 MLDv2 Report messages with Multicast Address Record TO_EX (NULL) for M2.
Step 25: The HUT must transmit MLDv2 Report message with Multicast Address Record IS_EX (S1) for M1.
Step 27: The HUT must transmit MLDv2 Report message with Multicast Address Record IS_EX (NULL) for M2.
