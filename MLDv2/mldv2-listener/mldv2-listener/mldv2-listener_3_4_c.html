Test Label

MLD.3.4: Listening State Update by One of Sockets
Part C: From INCLUDE (A) with INCLUDE (B) to EXCLUDE (NULL) with INCLUDE (B)

Purpose

To verify that the per-interface state is (re)evaluated when a service interface invocation modifies the socket status.

References

RFC3810 4.2. Per-Interface State
RFC3810 6.1. Action on Change of Per-Interface State

Discussion

None

Test Requirement

None

  • Common Test Setup
    1. Configure HUT to have:
      • Robustness Variable of 2
      • Query Interval of 125
      • Query Response Interval of 10 seconds
      • Unsolicited Report Interval of 1 second

  • Common Test Cleanup
    1. Return all timers and variables to their defaults values.
    2. Stop any listening. HUT does not listen to any multicast other than a solicited-node multicast address of an address assigned to the interface.

Procedure

Step 15. Enable MLDv2 on the HUT.
Step 16. Call ListenerAPI (S1, I1, M1, INCLUDE, {S1, S3}) on the HUT.
Step 17. Observe the packets on all networks.
Step 18. Call ListenerAPI (S2, I1, M1, INCLUDE, {S2, S3}) on the HUT.
Step 19. Observe the packets on all networks.
Step 20. Call ListenerAPI (S1, I1, M1, EXCLUDE, {NULL}) on the HUT.
Step 21. Observe the packets on all networks.

Observable Results

Step 17: The HUT must transmit 2 MLDv2 Report messages with Multicast Address Record ALLOW (S1, S3) for M1.
Step 19: The HUT must transmit 2 MLDv2 Report messages with Multicast Address Record ALLOW (S2) for M1.
Step 21: The HUT must transmit 2 MLDv2 Report messages with Multicast Address Record TO_EX (NULL) for M1.
