Test Label

MLD.4.9: MLDv2 Multicast Address and Source Specific Query Competition
Part B: EXCLUDE (NULL)

Purpose

To verify that the MLDv2 Listener replies an optimized report when queries are received continuously.

References

RFC3810 6.2. Action on Reception of a Query

Discussion

None

Test Requirement

None

  • Common Test Setup
    1. Configure HUT to have:
      • Robustness Variable of 2
      • Query Interval of 125
      • Query Response Interval of 10 seconds
      • Unsolicited Report Interval of 1 second

  • Common Test Cleanup
    1. Return all timers and variables to their defaults values.
    2. Stop any listening. HUT does not listen to any multicast other than a solicited-node multicast address of an address assigned to the interface.

Procedure

Step 7. Enable MLDv2 on the HUT.
Step 8. Call ListenerAPI (S1, I1, M1, EXCLUDE, {NULL}) on the HUT.
Step 9. Observe the packets on all networks.
Step 10. TR1 transmits MLDv2 Multicast Address and Source Specific Query message A for M1 (S1) with Maximum Response Code of 20000.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TR1 Link-local Address
        IPv6 Destination Address        = M1
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLD Query)
        Maximum Response Code           = 20000
        Multicast Address               = M1
        S Flag                          = 0
        QRV                             = 2
        QQIC                            = 125
        Number of Sources               = 1
        Source Address                  = S1
    -----
Step 11. TR1 transmits MLDv2 Multicast Address and Source Specific Query message B for M1 (S2) with Maximum Response Code of 1000.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TR1 Link-local Address
        IPv6 Destination Address        = M1
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLD Query)
        Maximum Response Code           = 1000
        Multicast Address               = M1
        S Flag                          = 0
        QRV                             = 2
        QQIC                            = 125
        Number of Sources               = 1
        Source Address                  = S2
    -----
Step 12. Observe the packets on all networks.

Observable Results

Step 9: The HUT must transmit 2 MLDv2 Report messages with Multicast Address Record TO_EX (NULL) for M1.
Step 12: The HUT must transmit MLDv2 Report message with Multicast Address Record IS_IN (S1, S2) for M1 within 1 second from Step 11.
         Then, the HUT must not transmit MLDv2 Report message with Multicast Address Record for M1 within 20 seconds from Step 10.

Possible Problems

Since HUT decides on time to transmit each MLDv2 Report message at random, the expected result may not be obtained. The expected result by several trials will be obtained. Besides, probability may be raised by adjusting Maximum Response Code of MLDv2 Query message.
