Test Label

MLD.2.7: Query Router Alert Option
Part A: No Hop-by-Hop Option

Purpose

To verify that an MLDv2 router properly processes Router Alerts in MLD Query messages.

References

RFC3810 7.6. Action on Reception of Queries

Discussion

None

Test Requirement

None

  • Common Test Setup
    1. Configure RUT to have:
      • Robustness Variable of 2
      • Query Interval of 125 (seconds)
      • Query Response Interval of 10 (seconds)
      • Last Listener Query Interval of 1 (second)
    2. Configure TR1, TR2, TR3 and TR4 to have:
      • Robustness Variable of 2
      • Query Interval of 125 (seconds)
      • Query Response Interval of 10 (seconds)
      • Last Listener Query Interval of 1 (second)

  • Common Test Cleanup
    1. Disable MLD on all devices.
    2. Return all timers and variables to their defaults values.
    3. The Query Interval of RUT can be shortened to decrease time.

Procedure

Step 1. TR1 has a lower IP address then the RUT.
Step 2. Enable MLDv2 on the RUT.
Step 3. Observe the packets on all networks.
Step 4. TR1 transmits Query A without a Hop-by-Hop Option.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TR1 link-local address
        IPv6 Destination Address        = All Nodes Address
    ICMPv6 Header (MLD Query)
        Maximum Response Code           = 10000
        Multicast Address               = ::
        S Flag                          = 0
        QRV                             = 2
        QQIC                            = 125
        Number of Sources               = 0
    -----
Step 5. Observe the packets on all networks.
Step 6. TN1 transmits Report A.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TN1 link-local address
        IPv6 Destination Address        = All MLDv2-capable routers
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLDv2 Report)
        Number of Mcast Address Records = 1
        Multicast Address Record [1]
            Type                            = 1 (IS_IN)
            Num of Sources                  = 1
            Multicast Address               = M1
            Source Address                  = S1
    -----
Step 7. Observe the packets on all networks.
Step 8. TR1 transmits Query B without a Hop-by-Hop Option.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TR1 link-local address
        IPv6 Destination Address        = M1
    no Hop-by-Hop Options Header (unusual)
    ICMPv6 Header (MLD Query)
        Maximum Response Code           = 1000
        Multicast Address               = M1
        S Flag                          = 0
        QRV                             = 2
        QQIC                            = 125
        Number of Sources               = 1
        Multicast Source Address        = S1
    -----
Step 9. Observe the packets on all networks.
Step 10. TN1 transmits Report B.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TN1 link-local address
        IPv6 Destination Address        = All MLDv2-capable routers
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLDv2 Report)
        Number of Mcast Address Records = 1
        Multicast Address Record [1]
            Type                            = 3 (TO_IN)
            Num of Sources                  = 0
            Multicast Address               = M1
    -----
Step 11. Observe the packets on all networks.

Observable Results

Step 3: The RUT must transmit 2 MLDv2 General Queries.
Step 5: The RUT must not crash or generate invalid packets.
        The RUT must not transmit Queries other then periodic General Queries.
Step 7: The RUT must not transmit Queries other then periodic General Queries.
Step 9: The RUT must not crash or generate invalid packets.
        The RUT must not transmit Queries other then periodic General Queries.
Step 11: The RUT must transmit 2 Multicast Address and Source Specific Query with a Multicast Address of M1 and a Source Address of S1.
