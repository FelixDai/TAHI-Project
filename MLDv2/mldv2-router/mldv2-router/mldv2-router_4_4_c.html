Test Label

MLD.4.4: Including - Receives Is Exclude and Updates Timer
Part C: After first source timer expires, Check Request List

Purpose

To verify that an MLDv2 router properly processes Is Exclude while in Is Include and updates the timer.

References

RFC3810 7.2.3. Definition of Source Timers
RFC3810 7.4.1. Reception of Current State Records
RFC3810 7.5. Switching Router Filter Modes

Discussion

None

Test Requirement

None

  • Common Test Setup
    1. Configure RUT to have:
      • Robustness Variable of 2
      • Query Interval of 125 (seconds)
      • Query Response Interval of 10 (seconds)
      • Last Listener Query Interval of 1 (second)
    2. Configure TR1, TR2, TR3 and TR4 to have:
      • Robustness Variable of 2
      • Query Interval of 125 (seconds)
      • Query Response Interval of 10 (seconds)
      • Last Listener Query Interval of 1 (second)

  • Common Test Cleanup
    1. Disable MLD on all devices.
    2. Return all timers and variables to their defaults values.
    3. The Query Interval of RUT can be shortened to decrease time.

Procedure

Step 19. Enable MLDv2 on the RUT.
Step 20. Observe the packets on all networks.
Step 21. TN1 transmits Report A.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TN1 link-local address
        IPv6 Destination Address        = All MLDv2-capable routers
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLDv2 Report)
        Number of Mcast Address Records = 1
        Multicast Address Record [1]
            Type                            = 1 (IS_IN)
            Num of Sources                  = 2
            Multicast Address               = M1
            Source Address                  = S1
            Source Address                  = S3
    -----
Step 22. Observe the packets on all networks.
Step 23. Wait [QI] seconds after Step 21.  TN1 transmits Report B.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TN1 link-local address
        IPv6 Destination Address        = All MLDv2-capable routers
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLDv2 Report)
        Number of Mcast Address Records = 1
        Multicast Address Record [1]
            Type                            = 2 (IS_EX)
            Num of Sources                  = 2
            Multicast Address               = M1
            Source Address                  = S2
            Source Address                  = S3
    -----
Step 24. Observe the packets on all networks.
Step 25. Wait [MALI] seconds after Step 21.  TN1 transmits Report D.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TN1 link-local address
        IPv6 Destination Address        = All MLDv2-capable routers
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLDv2 Report)
        Number of Mcast Address Records = 1
        Multicast Address Record [1]
            Type                            = 3 (TO_IN)
            Num of Sources                  = 0
            Multicast Address               = M1
    -----
Step 26. Observe the packets on all networks.

Observable Results

Step 20: The RUT must transmit 2 MLDv2 General Queries.
Step 22: The RUT must not transmit Queries other then periodic General Queries.
Step 24: The RUT must not transmit Queries other then periodic General Queries.
Step 26: The RUT must transmit 2 Multicast Address Specific Query with a Multicast Address of M1.
