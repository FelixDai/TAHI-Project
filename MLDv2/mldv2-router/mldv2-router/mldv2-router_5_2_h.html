Test Label

MLD.5.2: MLDv1 Report in MLDv2 Mode
Part H: Excluding Source with Requested Another Source, MLDv1 Done

Purpose

To verify that an MLDv2 router properly process an MLDv1 Query while in MLDv2 mode.

References

RFC3810 8.3.2. In the Presence of MLDv1 Multicast Address Listeners
RFC3810 9.13. Older Version Host Present Timeout

Discussion

None

Test Requirement

None

  • Common Test Setup
    1. Configure RUT to have:
      • Robustness Variable of 2
      • Query Interval of 125 (seconds)
      • Query Response Interval of 10 (seconds)
      • Last Listener Query Interval of 1 (second)
    2. Configure TR1, TR2, TR3 and TR4 to have:
      • Robustness Variable of 2
      • Query Interval of 125 (seconds)
      • Query Response Interval of 10 (seconds)
      • Last Listener Query Interval of 1 (second)

  • Common Test Cleanup
    1. Disable MLD on all devices.
    2. Return all timers and variables to their defaults values.
    3. The Query Interval of RUT can be shortened to decrease time.

Procedure

Step 63. Enable MLDv2 on the RUT.
Step 64. Observe the packets on all networks.
Step 65. TN1 transmits Report D.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TN1 link-local address
        IPv6 Destination Address        = All MLDv2-capable routers
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLDv2 Report)
        Number of Mcast Address Records = 1
        Multicast Address Record [1]
            Type                            = 2 (IS_EX)
            Num of Sources                  = 1
            Multicast Address               = M1
            Source Address                  = S1
    -----
Step 66. Observe the packets on all networks.
Step 67. TN1 transmits Report A.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TN1 link-local address
        IPv6 Destination Address        = All MLDv2-capable routers
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLDv2 Report)
        Number of Mcast Address Records = 1
        Multicast Address Record [1]
            Type                            = 1 (IS_IN)
            Num of Sources                  = 1
            Multicast Address               = M1
            Source Address                  = S1
    -----
Step 68. Observe the packets on all networks.
Step 69. TN1 transmits Report C.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TN1 link-local address
        IPv6 Destination Address        = M1
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLDv1 Done)
        Multicast Address               = M1
    -----
Step 70. Observe the packets on all networks.
Step 71. Wait [LLQI] seconds.  TN1 transmits Report E.
    -----
    IPv6 Header
        IPv6 Hop Limit                  = 1
        IPv6 Source Address             = TN1 link-local address
        IPv6 Destination Address        = All MLDv2-capable routers
    Hop-by-Hop Options Header
        Router Alert Option
    ICMPv6 Header (MLDv2 Report)
        Number of Mcast Address Records = 1
        Multicast Address Record [1]
            Type                            = 3 (TO_IN)
            Num of Sources                  = 0
            Multicast Address               = M1
    -----
Step 72. Observe the packets on all networks.

Observable Results

Step 64: The RUT must transmit 2 MLDv2 General Queries.
Step 66: The RUT must not transmit Queries other then periodic MLDv2 General Queries.
Step 68: The RUT must not transmit Queries other then periodic MLDv2 General Queries.
Step 70: The RUT must transmit 2 MLDv2 Multicast Address and Source Specific Query with a Multicast Address of M1 and a Source Address of S1.  
         The RUT must transmit 2 MLDv2 Multicast Address Specific Query with a Multicast Address of M1.
Step 72: The RUT must not transmit Queries other then periodic MLDv2 General Queries.
