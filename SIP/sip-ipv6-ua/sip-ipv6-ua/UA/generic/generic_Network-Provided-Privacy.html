 Generic judgement items for Network-Provided Privacy
   
   Should use network or transport layer security when contacting a privacy service. [RFC3323-4-20]
   Should establish a direct connection to a privacy service. [RFC3323-4-21]
   Should use lower-layer security mechanism to contact a privacy service 
                            when a direct connection is impossible. [RFC3323-4-22]
 
 - Headers:
 
   * Privacy
       Should include a Privacy header when network-provided privacy is required. [RFC3323-4-10]
       Should not use "user" level privacy for requests except REGISTER. [RFC3323-4-14]
       Must not remove or alter a Privacy header whose priv-value is 'none'. [RFC3323-4-16]
       A privacy header must consist of either the value 'none', 
                  or one or more of the values 'user', 'header', 'session'. [RFC3323-4-17]
       A privacy header must include the same priv-value at most once. [RFC3323-4-18]
       Must remove Privacy header from the message when it contain no priv-values except 'critical'. [RFC3323-5-9]
 
   * Proxy-Require
       Should include a Proxy-Require header containing 'privacy' option-tag. [RFC3323-4-23]
       Must remove any 'privacy' option-tag from Proxy-Require header 
                             when the entire Privacy header was removed. [RFC3323-5-10]
