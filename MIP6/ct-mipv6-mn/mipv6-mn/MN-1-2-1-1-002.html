SETUP

  • Common Setup-1

      

    • Reboot NUT

    • Enable MN function

        

      • Turn on MN functions

      

    • Initialize IPsec configuration

    • Set IPsec configuration

        

      • This test requires following configurations.

          

        • MN-HA0 SA1/SA2, Transport mode, Binding Update/Binding Acknowledgement

        

      • This test requires following configurations, If a related message is used.

          

        • MN-HA0 SA3/SA4, Tunnel mode, Return Routability Signaling

        • MN-HA0 SA5/SA6, Transport mode, Prefix Discovery

        

      • This test also allow to use following configurations.

          

        • MN-HA0 SA7/SA8, Tunnel mode, Payload Packets

        • MN-CN0, Transport mode, ICMP

        

      

    • Set IKEv1 configuration

        

      • Not using the optional KE payload.

          

        • w/o Perfect Forward Secrecy (PFS) in IKEv1 Phase 2

        

      • MN must be the initiator of the Security Association.

      • MN should establish required IPsecSA as an initiator after ISAKMP SA establishment.

          

        • Be established within FUNC_DETAIL_IKE_TIME in config (default 30sec). 

        • Since a test is not made when not establishing SA, it is ended by Initialization Fail.

        

      • Interval when re-establishment of each SA recommended while testing begins.
(It is necessary to be likely to adjust it more.)

          

        • ISAKMP SA     : 480 sec

        • IPsec SA1/SA2 : 240 sec

        • IPsec SA3/SA4 : 120 sec

        • IPsec SA5/SA6 : 120 sec

        • IPsec SA7/SA8 : 120 sec

    TEST PROCEDURE

                        
       HA0    NUTX      R1      R2     CN0
        |       |       |       |       |
        |       | <---- |       |       | 1.Router Advertisement
        |       |       |       |       |
        | <===> |       |       |       | a.IKEv1 Phase1 (ISAKMP SA)
        | <===> |       |       |       | b.IKEv1 Phase2 (IPsec SA1/SA2)
        |       |       |       |       |
        | <---- |       |       |       | 2.Binding Update
        | ----> |       |       |       | 3.Binding Acknowledgement
        |       |       |       |       |
        |       |       |       |       | :
        |       |       |       |       |
        |       |       |       |       | c.(expire ISAKMP SA)
        | <===> |       |       |       | d1.IKEv1 Phase1 (ISAKMP SA)
        |       |       |       |       |
        |       |       |       |       | :
        |       |       |       |       |
        |       |       |       |       | e.(expire IPsec SA1/SA2)
        | <===> |       |       |       | d2.IKEv1 Phase1 (ISAKMP SA)
        | <===> |       |       |       | f.IKEv1 Phase2 (IPsec SA1/SA2) (*1)
        |       |       |       |       |
        | <---- |       |       |       | 4.Binding Update (*2)
        | ----> |       |       |       | 5.Binding Acknowledgement
        |       |       |       |       |
        | ----> |       |       |       | 6.ICMP Echo Request
        | <---- |       |       |       | 7.ICMP Echo Reply (*3)
        |       |       |       |       |
                    
        1. Send Router Advertisement. (R1 -> R1_allnode_multi)
                    
        a. IKEv1 Phase1 (ISAKMP SA)
        b. IKEv1 Phase2 (IPsec SA1/SA2)
                    
        2. Receive Binding Update. (NUTX -> HA0)
        3. Send Binding Acknowledgement. (HA0 -> NUTX)
                    
        c. (expire ISAKMP SA)
       d1. IKEv1 Phase1 (ISAKMP SA) or [d2]
        e. (expire IPsec SA1/SA2)
       d2. [d1] or IKEv1 Phase1 (ISAKMP SA)
        f. IKEv1 Phase2 (IPsec SA1/SA2)
                    
        4. Receive Binding Update. (NUTX -> HA0)
        5. Send Binding Acknowledgement. (HA0 -> NUTX)
        6. Send ICMP Echo Request. (HA0 -> NUTX with Type2 Routing Header)
        7. Receive ICMP Echo Reply. (NUTX -> HA0 with Home Address Option)
                    
    Focus Packet Format:
        4. Binding Update

    JUDGEMENT

     (*1) PASS: IPsec SA1/SA2 is re-established after re-establishing ISAKMP SA.
 (*2) PASS: HA0 receives Binding Update.
            Then, check whether this packet fills all of the following,
            - using new IPsec SA1.
 (*3) PASS: HA0 receives ICMP Echo Reply with Home Address Option.
